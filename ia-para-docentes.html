Bloque 1 · Contexto

El nuevo rol del docente en la era de la IA

Durante años, el equipo docente ha invertido horas en crear materiales desde cero: temarios, actividades, exámenes, rúbricas. La IA no elimina ese trabajo — lo transforma.

El nuevo rol no es creador, sino curador y revisor: describes lo que necesitas, la IA genera un borrador y tú lo ajustas, mejoras y validas con tu criterio profesional.

💡 Idea clave: La calidad final depende del docente, no de la IA. La IA acelera el primer borrador; el juicio humano garantiza la calidad.

Hoy vamos a construir juntos un ecosistema real: desde la nota en Markdown hasta una página web publicada y accesible para cualquier alumno.

📝 Obsidian → 🤖 Cowork → 💻 Claude Code → 🐙 GitHub → 🌐 Vercel

Bloque 2 · Fundamento técnico

Markdown: el idioma que conecta todo

Markdown (.md) es texto plano con una sintaxis mínima que cualquier editor entiende. Es el hilo conductor de todo el ecosistema.

  • # Título → encabezado principal
  • **negrita** → texto en negrita
  • - elemento → lista con viñetas
  • [texto](url) → enlace

¿Por qué importa para trabajar con IA?

  • Menos tokens = menor coste al enviar contenido a la IA.
  • Mayor compatibilidad: GitHub, Obsidian, Notion, VS Code, correo… todos lo entienden.
  • Revisión humana sencilla: el docente puede leer y editar sin software especial.
  • Base para la web: Claude Code convierte cualquier .md en HTML en segundos.
🔗 Regla de oro: Todo nace como .md en Obsidian. A partir de ahí, la IA puede convertirlo en lo que necesites.

Bloque 3 · Cowork y Skills

Claude Cowork: tu asistente dentro de Obsidian

Claude Cowork es un asistente de IA integrado directamente en tu Vault de Obsidian. Entiende tus notas, tu estructura y puede crear, modificar y enlazar documentos sin salir del editor.

¿Qué son los Skills?
Los Skills son capacidades especializadas que amplían lo que puede hacer Cowork. Cada skill tiene su propio conjunto de instrucciones optimizadas para una tarea docente concreta.

  • 📚 Temario didáctico — genera unidades de aprendizaje con objetivos Bloom.
  • 📋 Actividades + rúbrica — diseña actividades con criterios de evaluación.
  • 📝 Generador de exámenes — crea pruebas balanceadas desde el temario.
  • ✅ Corrector de alumnos — evalúa respuestas y genera feedback constructivo.
  • 🌐 Lección interactiva — convierte cualquier .md en una web interactiva.
🎯 Flujo típico: Describes el módulo → Cowork genera el borrador → tú revisas y ajustas en lenguaje natural → material listo en minutos.

Bloque 4 · Claude Code

Claude Code: el agente que actúa en tu repositorio

Claude Code no es solo un chat — es un agente que puede leer, escribir y ejecutar comandos directamente en tu proyecto. Vive en la terminal.

¿En qué se diferencia de Cowork?

  • Cowork trabaja dentro de Obsidian, sobre documentos y notas.
  • Claude Code actúa sobre el repositorio de código: genera HTML, ejecuta git, despliega a Vercel.

¿Qué puede hacer en la sesión de hoy?

  • Leer un .md y convertirlo en una página web interactiva completa.
  • Ajustar el diseño en lenguaje natural: "usa los colores de marca", "añade una barra de progreso".
  • Subir el resultado a GitHub con git init, git add, git commit, git push.
⚡ Principio clave: Tú describes el resultado en español. Claude Code traduce eso a código y comandos. No necesitas saber programar.

Bloque 5 · Publicación

GitHub + Vercel: de archivo local a URL pública

Una vez que Claude Code genera el material, dos herramientas lo llevan al mundo real en cuestión de minutos.

GitHub — la nube del código

  • Almacena el proyecto en la nube de forma gratuita.
  • Permite colaboración: varios docentes pueden trabajar sobre el mismo repositorio.
  • Guarda el historial de cambios: puedes volver a cualquier versión anterior.

Vercel — el despliegue automático

  • Se conecta a GitHub con 3 clics.
  • Cada vez que hay un cambio en GitHub, Vercel despliega la nueva versión automáticamente.
  • El material queda en una URL pública accesible desde cualquier dispositivo.
🌟 El momento "wow": Haces un cambio en el texto, Claude Code lo sube a GitHub… y en 30 segundos la web ya está actualizada para todos los alumnos.
